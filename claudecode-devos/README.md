# ClaudeCode DevOS Initial Set

This directory is the reviewable source tree for `/opt/claudecode-devos`.

## Install

```bash
sudo SERVICE_USER=kensan ./install.sh
python3 -m pip install --user -r /opt/claudecode-devos/requirements.txt
sudo apt install msmtp msmtp-mta mailutils
gh auth login
sudo systemctl enable --now memory-guard.timer
sudo systemctl enable --now usage-manager.timer
sudo systemctl enable --now claudecode-session.timer
# Optional GitHub automation loop:
# sudo systemctl enable --now github-loop.service
# Optional decision/CI orchestration:
# sudo systemctl enable --now autonomous-orchestrator.timer
# Optional dashboard:
# sudo systemctl enable --now devos-dashboard.service
# Optional reports:
# sudo systemctl enable --now devos-report.timer
# Optional self-evolution:
# sudo systemctl enable --now devos-evolution.timer
# Optional cluster controller:
# sudo systemctl enable --now cluster-orchestrator.timer
# sudo systemctl enable --now cluster-requeue.timer
# Optional cluster worker:
# sudo systemctl enable --now worker-heartbeat.timer worker-poll-jobs.timer
# Optional strategy scoring:
# sudo systemctl enable --now strategy-cycle.timer
# Recommended log retention:
# sudo systemctl enable --now log-retention.timer
```

## Notes

- The systemd units use `User=kensan` by default. Override with `sudo SERVICE_USER=<user> ./install.sh`.
- `run-session.sh` composes `Docs/session_context.md`, `state.json`, and the project `START_PROMPT.md` into a single prompt before launching Claude through `claude-safe.sh`.
- `recovery.sh` uses a lock to avoid duplicate recovery runs and targets the recorded Claude PID before falling back to broad process handling.
- `usage_manager.py` enforces the daily 5-hour and weekly 90,000-second limits through `state.json`.
- Gmail alerts use `mail` via msmtp when `ALERT_MAIL_ENABLED=true` and `ALERT_MAIL_TO` are configured.
- GitHub automation uses `gh` and feature branches. Automatic merge is disabled unless `GITHUB_AUTO_MERGE=true`.
- The autonomous orchestrator chooses `develop`, `repair_ci`, `cooldown`, or `suspend` from `state.json`.
- Prompt generation composes state, selected issue, CI summary, and DevOS Docs before auto development runs.
- The project scheduler selects one active project by weighted priority and due date.
- The Flask dashboard exposes `/`, `/api/state`, and `/api/projects`.
- Dashboard manual controls can request `develop`, `repair_ci`, `cooldown`, `suspend`, or `resume`.
- Dashboard project editing can update `priority`, `status`, and `weight`.
- WorkTree mode isolates automated development and CI repair from the main worktree.
- Markdown reports are generated by `reports/report_generator.py`; generated `report_*.md` files are local artifacts.
- The evolution loop records outcomes, analyzes metrics, and writes prompt guidance for future runs.
- Cluster mode uses JSON job files, worker heartbeats, and optional SSH/rsync sync with `CLUSTER_SYNC_ENABLED=true`.
- Cluster failover uses a simple file leader. Set `CLUSTER_CONTROLLER_ID` per controller before enabling the cluster timer.
- Strategy scoring ranks projects by ROI, urgency, reuse, stability, and personal interest.
- Core operational policies are under `docs/12_policies`, with schemas in `config/*.schema.json`.
- Config invariants are checked by `ops/validate_config.py`.
- Log retention is handled by `ops/log_retention.sh` and `log-retention.timer`.
