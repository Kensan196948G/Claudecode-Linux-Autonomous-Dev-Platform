# ClaudeCode DevOS レポートテンプレート

このテンプレートは `reports/report_generator.py` が生成するレポートの構成を定義します。
各項目は、あとから作業状況を追跡しやすいように、短く具体的に記録してください。

## 記載項目

- 時刻: レポート生成時刻、対象期間、セッション開始からの経過時間を記録します。
- 判断: 現在のモード、次のアクション、判断理由を記録します。
- 使用量: 日次/週次の実行時間、トークンまたは利用制限に関する状態を記録します。
- リソース: メモリ、swap、CPU、ディスク使用率など、実行継続に影響する状態を記録します。
- CI: 最新CIの結果、失敗しているジョブ、修復要否、STABLE判定、Codexレビュー、security確認を記録します。
- スケジューラ: 選択されたプロジェクト、次回実行予定、スキップ理由を記録します。
- AI判断ログ: Monitor、Build、Verify、Improve、Repairなどの判断履歴を記録します。
- プロジェクト実行履歴: 実行したプロジェクト、ブランチ、WorkTree、PR、結果を記録します。
- CI修復履歴: 修復対象、仮説、修正内容、再実行結果、リトライ回数を記録します。
- プロジェクト一覧: 各プロジェクトの状態、優先度、Issue、CI、最終実行結果を表形式で記録します。

## 5時間上限

- 自動開発セッションは最大5時間で終了します。
- 残り30分を切ったら新しい大きな作業を開始せず、検証と整理を優先します。
- 残り15分を切ったらVerifyを縮退し、終了準備を開始します。
- 残り5分を切ったら即時に終了処理へ移行します。
- 終了時は、commit、push、PR、CI結果、残課題、再開ポイントを必ず記録します。

## 分かりやすく書くための基準

- 「何をしたか」「なぜそう判断したか」「次に何をするか」を分けて書きます。
- エラーは原因、影響範囲、対応方針をセットで書きます。
- CIが未確認の場合は、成功扱いにせず「未確認」と明記します。
- STABLE判定は、test、lint、build、CI、review、security、error 0 を満たした場合のみ記録します。
- CodexやMemoryが未接続の場合は成功扱いにせず、「未接続」「未確認」と明記します。
- Agent Teamsを使った場合は、CTO、ProductManager、Architect、Developer、Reviewer、QA、Security、DevOpsなどの判断を短く分けて記録します。
