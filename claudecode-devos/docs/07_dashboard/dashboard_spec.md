# Dashboard Specification

## Purpose
- Make `state.json` readable at a glance.
- Show resource, CI, project, and recovery state on one page.
- Show multi-project scheduler output.

## Main Panels
- System Status
- Next Action
- Resources
- CI Status
- Usage
- Selected Project
- Queued Projects
- Decision Log
- Recovery Log
- Project Edit UI

## Editable Project Fields
- priority
- status
- weight

## Notes
- This dashboard prioritizes lightweight state visibility.
- Charts can be added later after the state surface is stable.
